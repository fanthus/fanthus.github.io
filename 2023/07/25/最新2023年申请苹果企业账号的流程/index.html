最新(2023)年申请苹果企业账号的流程

Tue Jul 25 2023
  • iOS

# 申请 AppleID

申请 AppleID 的时候，邮箱最好用自己的公司邮箱。

如果之前你公司邮箱的 AppleID 已经成为了某个企业开发者账号的 Owner 的话，最好换一个 AppleID 申请。

# 申请邓白氏编码

看这篇文章 2023年申请邓白氏编码流程 (opens new window)

邓白氏编码申请完之后需要等几天才能继续申请苹果账号，因为邓白氏向苹果公司上传 DUNS 信息需要一些时间，我差不多等了四天才开始继续后面的流程。

# 申请苹果账号

总的流程基本是按照官网以组织身份注册 Apple Developer Program (opens new window)这篇文章来进行。

# 准备工作

  1. 苹果账号准备。 我的账户已经是一个公司的账户持有人了，所以使用同事的邮箱重新进行注册了一个苹果站账号来进行下一步的申请。
  2. 设备和软件准备。 在用于注册的设备上，下载 Apple Developer App。当前设备需要开启密码/面容/指纹任意一个认证方式。不要使用已经申请过开发者账户的设备再次注册。

# 开始注册

# 注册入口

启动 Apple Developer App，并切到 “账户”标签页(tab)。查看 Apple Developer Program 区域里的立即注册按钮。如果不能点击检查一下第一步里是不是开了设备认证权限。

点击立即注册，开始填信息。

# 填写信息

官方有如下提示

输入你的名字、姓氏、身份证号和电话号码。请勿输入别名、昵称或公司名称作为你的姓氏或名字，因为这会导致注册审核延误。

不过刚开始的界面不需要填身份证，主要是姓名和地址。我填写了我的名字和公司的地址。

重点来了，我在填写信息这儿卡住了，填写完信息之后一直报错，提示「请输入一个有效的地址」。 发现怎么改都不对，给苹果客服 (opens new window)打了个电话问了问，说有详细的格式要求，客服会给我们发邮件说明。

我填写的信息里名字是我的，苹果账号是同事的，我联系苹果客服的时候还需要同事在场授权。 PS: 其实应该把信息里的名字和苹果账号对应起来，这样后续会少很多不必要的麻烦。

等了几分钟之后，收到下面得邮件内容，这邮件发了跟没发差不多..

重新调整了几次之后提交还是不行，最后一次提交后，App 提示让我联系苹果开发者团队来继续审核流程。

重新联系客服之后，说还是地址有问题，需要我们提交一些个人相关的证件信息来审核，会给我们发邮件具体说明，然后我们按要求提交了相关信息，继续等待。

  1. 到这一步 Apple Developer App 里面的「现在注册」按钮已经点不动了，只能眼巴巴等待苹果审核信息。
  2. 这时候似乎不能再通过电话联系苹果客服团队了，联系客服网页会提示用邮箱进行沟通。

等了一会儿，收到邮件提示说我们提交的证件地址信息和填写的地址信息不一样，我突然意识到，可能地址栏里不需要填公司地址，填个人地址也行。

于是同事重新填了个人的地址提交审核过了，让我意外的是地址信息的两栏并不需要完全填全，填一栏就能通过审核了（咱也不知道苹果这个地址填写规则到底是咋回事）。

接下来就是提交身份证信息、实体类型（选组织），以及组织详情（英文）。组织详情的部分需要填写之间申请好的邓白氏编码。

最后是继续补充公司的的一些信息，以及能够确认你身份的公司联系人的信息等，这儿的联系信息要填正确，后面是真的会联系的。

填写信息流程搞完之后，会收到提示「您的注册正在审核中」。

接下来会收到一封邮件如图，表示收到你的申请了，等着审核吧。

这时候提醒一下你留联系方式的公司同事，他的邮箱可能会收到苹果的邮件，来确认相关信息。

# 确认信息

大概几个小时之后，另外一个同事（之前填写的公司联系人）邮箱会收到一封信息，指导同事点击邮箱中的链接、并继续填写信息。苹果安排这步主要就是确认信息，保证之前填的信息都是真实的。

# 完成注册

信息填完之后再回到 Apple Developer App 会发现之前的审核状态已经结束了，现在提示「继续您的注册流程」。

点击继续注册后，查看《Apple Developer Program 许可协议》 (opens new window) 的条款并轻点或点按“同意”。这时候查看你的年度会员资格订阅详情，然后轻点“以每年 ¥688 续订”。

但是我进行到这一步的时候又出问题了，点击续订后，界面弹窗「已订阅会员资格续订」… 我怀疑是因为当前设备登录了之前开发者的 iCloud 账号，于是退出 iCloud，然后 iCloud 登录当前开发者 Apple ID，并且绑定了支付宝支付。重启手机后，再次点击续订，还是一模一样的提示。去苹果开发者后台 (opens new window)看也提示需要再手机端完成这个订阅操作。

实在没办法，又给苹果回了一封邮件，说了一下当前的问题，等待苹果回复，没等到回复。拨了电话回去，苹果开发者客服听完描述之后，主要确认的是当前下载 Apple Developer App 并进行企业账号注册的设备，之前是否给别的企业进行过类似的操作？并没有。然后客服要求我们提供下订阅出错的录屏演示，以及当前账号的 AppStore 购买记录。说实话，我感觉客服应该也没有搞清楚到底是怎么回事，于是我就又开始尝试几种不同的方法。

  1. 当前开发者的的苹果账号是新申请的，还没有购买过应用。尝试购买新应用的时候会要求同意 AppleStore 的协议，并确认付款方式。

    这一步做完之后去 Apple Developer App 再进行订阅发现还是提示 「已订阅会员资格续订」

  2. 卸载之前开发者苹果账号下载的 Apple Developer App，用当前开发者苹果账号下载 Apple Developer App，并在应用中登录当前开发者苹果账号。这时候再续订，发现已经能续订成功了。

方法二最终解决了「已订阅会员资格续订」的问题，Apple Developer App 显示当前账户已经是新公司主体的账户持有人了。

总结一下

  1. 不要使用已经注册过开发者的设备再次进行开发者注册。 苹果会校验两家公司的关联性，并要求提供营业执照、账号持有人身份证登很多信息。最后可能也会过，但是会增加注册流程和难度，没有必要。
  2. 注册开发者的苹果账号要独自在商店完成 Apple Developer App 的下载，并使用它来完成账号注册。 苹果会校验当前 Apple Developer App 的下载者是否注册过开发者计划（感觉还挺神奇的）

至此，苹果企业开发者账号已经申请完毕了。希望对大家有帮助。
